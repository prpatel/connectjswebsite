Jasmine and Cucumber - a Tasty way to Deliver Better Code Faster

Jasmine and Cucumber - a Tasty way to Deliver Better Code Faster

Behavior Driven Development is a proven way to deliver higher quality code to meet your business’ requirements in less time and you can add it to any agile processes (like Scrum) that does not specify how software should be developed. In this session we’ll look at two BDD frameworks; Cucumber and Jasmine. Cucumber is a tool that lets you turn user stories into executable specifications that run every time you check in a code change, so you can verify that what you’re delivering meets the business’ acceptance criteria. Jasmine can be used to write lower level specifications that describe the services needed to make the executable specifications pass. You can use it to write unit tests that ensure the code you’ve written behaves the way you expect. Using them together leads to some nice results. You only write code needed to meet the user acceptance criteria, your code behaves the way you expect it to, and you can safely refactor your code to improve clarity, simplify its structure, or meet any other metric you deem necessary. This focused development results in highly cohesive, loosely coupled code, which means it’s easier to debug, maintain and enhance. Now isn’t that a tasty treat?

Virtual reality (VR) is an amazing medium that helps create immersive stories for nearly any industry, from journalism to manufacturing to gaming. Virtual reality hardware systems like Google Cardboard, Oculus Rift, Samsung Gear and others are starting to making VR a household name. On the software side, most of the activity is confined to native applications, but the web can bring major advantages to VR that only increase the accessibility of this platform, including experiencing VR without downloading and installing apps and allowing seamless connections from one experience to another via hyperlinks. This session will examine the technologies bringing VR to the web including Google’s VR view & VR shell, Mozilla VR and WebVR, a vendor--vendor-independent web standards for VR.

CSS from the Future

CSS from the Future

For years, we’ve relied on preprocessors to simplify CSS development, but the time is coming when we won’t need this anymore. Variables, mixins, mathematic calculations, color manipulation, and nested selectors are all in the official specification, at various stages of development. Nearly every feature provided by SASS and LESS is coming to the browser. Some of it has already arrived. Some of it is even more powerful than the preprocessor you are used to. In this talk, I’ll show you how all these features work. And I’ll show you how, using the PostCSS processor, you can transpile this future syntax into code you can use today. Come take a glimpse at what the future of CSS holds.
